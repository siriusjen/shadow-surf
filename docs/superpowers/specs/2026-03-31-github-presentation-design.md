# GitHub Presentation Design

**Goal:** 让 `ShadowSurf` 的 GitHub 仓库首页更偏产品表达，优先打动 IntelliJ / IDEA 用户，突出“减少白底割裂感”和“少切应用轻量浏览”两条主线。

## Audience
- 首要受众：IntelliJ / IDEA 用户
- 次要受众：对插件实现感兴趣的开发者

## Positioning
- `ShadowSurf` 不是完整浏览器
- 它是 IDEA 内的轻量浏览 Tool Window
- 核心价值：在不离开 IDE 的情况下，快速打开网页，并降低暗色 IDE 与白底网页之间的视觉割裂

## Deliverables
1. 重构 `README.md` 顶部结构
   - 顶部一句话价值描述
   - 亮点列表
   - 适合谁/不适合谁
   - 安装与快速上手前置
   - 开发说明下移
2. 新增 `CHANGELOG.md`
   - 记录 `0.1.0` MVP 内容、验证状态、已知限制
3. 补一份 GitHub 仓库信息文案
   - About description
   - Website / topics 建议
   - 可复制的仓库简介

## Content Style
- 偏产品表达，但不夸大能力
- 先讲“为什么值得用”，再讲“怎么安装”
- 强调 MVP 边界，避免误导为完整浏览器
- 文案简洁、直接、可扫描

## Constraints
- 不改产品方向
- 不新增功能
- 不虚构截图或能力
- 文案必须与当前已验证行为一致
